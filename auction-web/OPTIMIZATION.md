# 项目优化说明

## 优化概述
前端演示版聚焦拍卖列表/详情的搜索、筛选、列配置与出价交互，使用 Hooks 做基础性能优化，并将关键配置持久化到 `localStorage`。

## 主要优化内容

### 1. 代码与结构
- 组件拆分：PageHeader、TableToolbar、ResizableTable、ListPage、DetailPage
- 工具抽取：`storage.js`（localStorage 封装）、`filter.js`（搜索与台数统计）、`constants/index.js`（默认数据与配置）
- 路由：React Router 7.10.1，`/` 与 `/detail/:id`

### 2. 交互与体验
- 列配置：商品列必选；标单号默认隐藏；全选/清空；配置持久化
- 出价交互：历史出价作为 placeholder；Enter/按钮提交；>10000 弹确认；取消出价带确认；自动聚焦下一行
- 快捷键：↑/↓ 切换出价框并清空当前未提交值；Escape 清空当前输入
- 视觉：已出价行左侧绿色竖条，未出价透明竖条；表格与筛选面板保持响应式

### 3. 性能与可维护性
- useMemo/useCallback 缓存列定义、过滤结果、统计与事件
- 搜索 → 供应商 → 成色 → 出价状态的级联过滤，减少重复计算
- Vite 手动分包（react-vendor / antd-vendor），esbuild 压缩，cssCodeSplit 开启

## 优化前后对比（定性）
- 代码模块化，职责更清晰，样式与逻辑分离
- 交互反馈完善：无效输入处理、大额确认、取消确认、键盘操作
- 配置与出价持久化，刷新不丢失

## 技术债务
- 未对接后端 API（数据获取与出价提交/取消）
- 倒计时为静态文本，未实时更新
- 列宽不可拖拽或持久化
- 未添加单元测试/E2E
- 未记录最新构建体积（需本地 `npm run build`）

## 后续优化建议
- 接入后端接口与错误边界；按需引入 Ant Design 降低包体积
- 实时数据刷新与倒计时更新
- 导出/批量操作能力
- 补充单元测试与端到端测试
- 视需要增加多语言、主题切换、PWA 等增强能力
